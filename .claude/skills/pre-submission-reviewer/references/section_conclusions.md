# Subagent S-CO — Conclusions

You review only the **Conclusions**. Readers often go abstract → conclusion
before anything else, so the conclusion must be **comprehensible on its own**.
Evaluate it against the seven-element framework for effective conclusions.

## FRAMEWORK — the seven elements (Pols)

An effective conclusion:
1. can be read independently of the rest of the text;
2. restates the research question or goal;
3. reminds the reader of the study's context/relevance (the problem);
4. briefly describes the method;
5. answers the question (using the context and, where possible, reference to literature values);
6. provides substantiation for that answer;
7. provides the main recommendations and suggestions for future research.

Score N/7. Elements 1–3 make it independently readable; 4–7 make it complete.

## CHECKLIST (PASS / FAIL / PARTIAL, with location)

- `S-CO.1` Independently readable (element 1) — does not require having just read Results.
- `S-CO.2` Restates the question/goal (element 2) **without duplicating the introduction verbatim**.
- `S-CO.3` Reminds the reader of the context/problem (element 3).
- `S-CO.4` Briefly states the method (element 4).
- `S-CO.5` Answers the question with the main finding (element 5), referencing literature values where relevant.
- `S-CO.6` Substantiates the answer from the study's own results (element 6).
- `S-CO.7` Gives recommendations / future work (element 7).
- `S-CO.8` **No new results or content** appear only in the Conclusions.
- `S-CO.9` Does not overgeneralize beyond what the data support; claims use conditional language where warranted.
- `S-CO.10` Avoids opening with "In conclusion" / "To conclude"; avoids verbatim reuse of introduction or abstract sentences.
- `S-CO.11` The contribution stated here matches the abstract and introduction (feeds C1/C4).
